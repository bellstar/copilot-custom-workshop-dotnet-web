# Custom Agent

[前へ - ユニットテスト](../8_UnitTesting/README_JA.md) | [次へ - まとめ](../10_LessonsLearned/README_JA.md)

このステップでは、**Custom Agent（`.agent.md`）** を作成し、プロジェクト専用の AI アシスタントを構築します。Before/After の比較で Custom Agent の価値を実感してください。

---

## Custom Agent とは

Custom Agent は `.agent.md` ファイルで定義する「特化型 AI アシスタント」です。

| 項目 | Custom Instructions | Custom Agent |
|------|-------------------|--------------|
| 定義ファイル | `.github/copilot-instructions.md` | `.github/agents/xxx.agent.md` |
| 適用範囲 | すべてのチャット | `@agent名` で呼び出した時のみ |
| 用途 | コーディング規約・プロジェクト情報 | 特定ドメイン知識・判断ロジック |
| 参照できるもの | なし | 他のファイルを `#file` で含められる |

**使い分け:**
- Custom Instructions = 「全員が従うべきルール」
- Custom Agent = 「専門知識を持つチームメンバー」

---

## 1. アーキテクチャガイドの生成

まず、Custom Agent が参照するプロジェクト情報を Copilot に生成してもらいます。

Copilot Chat（Agent モード）で：

```
#codebase を分析して、docs/architecture-guide.md を作成してください。
以下を含めてください：
- プロジェクト構成（ディレクトリ・ファイル構造）
- 使用技術スタック
- 命名規約（実際のコードから読み取ったもの）
- データモデルの関係
- 新機能追加時の標準パターン（コントローラー、ビュー、テストの配置場所）
```

このファイルが Custom Agent のナレッジベースになります。

---

## 2. Before: Custom Agent なしで機能追加

まず、**Custom Agent を使わない状態**で機能追加を試みます。結果を覚えておいてください。

Agent モードで以下を入力：

```
猫のお気に入り登録機能を追加してください。
ユーザーがお気に入りボタンを押すと、その猫がお気に入りリストに追加されます。
```

**観察ポイント:**
- 既存のコード規約に従っているか？
- ファイルの配置場所は既存パターンと一致しているか？
- テストも含めて実装されたか？
- 日本語コメント・命名規約は守られたか？

> 結果に問題がなくても構いません。After との比較のために出力を記憶しておいてください。

---

## 3. Custom Agent の作成

### `.github/agents/meowworld-dev.agent.md` を作成

```markdown
---
description: "MeowWorld プロジェクトの機能開発を担当するエージェント。新機能の設計・実装・テストを一貫して行う。"
tools:
  - codebase
  - terminal
  - file
---
# MeowWorld 開発エージェント

あなたは MeowWorld（猫カフェ管理システム）の専任開発者です。

## プロジェクト知識

#file:docs/architecture-guide.md を必ず参照して、プロジェクトの構造と規約を理解した上で作業してください。

## 新機能の開発プロセス

新機能のリクエストを受けたら、以下の順序で作業してください：

### Step 1: 設計
- 必要なモデル変更を特定する
- 既存のモデル（Cat.cs）との関係を明確にする
- マイグレーションが必要か判断する

### Step 2: 実装
- モデル → DbContext → コントローラー → ビュー の順で実装する
- 既存コードのパターンに厳密に従う：
  - コントローラーは `Controllers/` 直下
  - ビューは `Views/{ControllerName}/` 配下
  - file-scoped namespace を使用
  - 非同期メソッドを使用
  - 日本語の XML ドキュメントコメント

### Step 3: テスト
- テストクラスは `MeowWorld.Tests/Controllers/` に配置
- テストメソッド名は日本語
- AAA パターン + InMemory DB
- 正常系 + 異常系（最低 3 ケース）を含める

### Step 4: 確認
- `dotnet build` でビルド確認
- `dotnet test` で全テストパス確認
- 既存テストが壊れていないか確認

## 制約
- 既存機能を壊さないこと
- 不必要なパッケージを追加しないこと
- 一度に複数のマイグレーションを作らないこと
```

---

## 4. After: Custom Agent で同じ機能追加

先ほどの変更を取り消すか、別ブランチで作業した上で、Custom Agent を使って同じ機能追加を行います。

Copilot Chat で `@meowworld-dev` と入力して Custom Agent を呼び出します：

```
@meowworld-dev 猫のお気に入り登録機能を追加してください。
ユーザーがお気に入りボタンを押すと、その猫がお気に入りリストに追加されます。
```

### Before と After の比較

| 観点 | Before（Agent のみ） | After（Custom Agent） |
|------|---------------------|---------------------|
| ファイル配置 | 規約に従わない場合がある | 必ず既存パターンに従う |
| テスト | 作成されないことがある | 必ず 3 ケース以上作成 |
| 実装順序 | 一括生成 | モデル→DB→コントローラー→ビュー→テストの順 |
| コード品質 | ばらつきがある | アーキテクチャガイドに厳密に準拠 |
| 既存テスト | 確認しないことがある | 全テストパスを確認 |

> **学びのポイント:** Custom Agent は「チームの熟練メンバーの知識」をコード化したものです。新メンバーが入っても、Custom Agent を通じて一貫した品質を維持できます。

---

## 5. Custom Agent を活用して機能を完成させる

Before/After の比較が終わったら、Custom Agent で生成した「お気に入り機能」を採用して実装を完成させます：

```
@meowworld-dev ビルドとテストが通ることを確認して、
何か問題があれば修正してください。
```

## 応用：他のエージェントのアイデア

同じ仕組みで様々な専門エージェントを作れます：

| エージェント名 | 用途 |
|--------------|------|
| `@reviewer` | コードレビューの観点でフィードバック |
| `@security` | セキュリティ観点のチェック |
| `@docs` | API ドキュメント / README の生成 |
| `@migration` | DB マイグレーション専門 |
